# 工程与测试约定

> 本文件是所有任务执行必须遵守的硬约束,来自用户明确要求。违反即返工。

## 产品命名与品牌标识

**产品名:一脉(Axon)**

- **一脉**:提交 → 扫描 → 部署 → 监控由 `service / git_sha / deployment_id` 三关联键串成一条可回溯链路,即设计文档所说"一脉贯穿";脉亦是输送通道,控制面是基础设施的命脉——控制指令下行、状态回流皆经此一脉。
- **Axon(神经元轴突)**:控制面把命令传导到每台纳管节点 / Agent 并回流状态,恰如基础设施的神经轴突,与"一脉"同为"体内贯通传导的通道"意象,中英同频。
- 命名克制、专业、无营销腔,契合前端设计规范「反 AI 感清单」。

| 文件 | 用途 |

## 用户五条硬约束(逐条落实)

1. **独立文件夹**:全部代码/文档收敛在 `统一运维控制面/` 下。
2. **前端参考 JumpServer**:配色、风格见 [前端设计规范.md](./前端设计规范.md)。
3. **不许一眼看穿是 AI 产物**:见前端设计规范「反 AI 感清单」,每页自检。
4. **每个功能做完做局部测试**:见下「局部测试」。
5. **测试与正式代码区分,禁止事后补 -test**:见下「测试时序与目录」。

## 目录结构

```
统一运维控制面/
├── docs/                     设计文档、任务拆分、本约定
├── backend/                  FastAPI 后端
│   ├── app/                  正式代码(api/ core/ models/ services/ adapters/ workers/)
│   └── tests/                测试代码(与 app/ 物理分离,镜像其结构)
│       ├── unit/
│       ├── integration/
│       └── conftest.py
└── frontend/                 React 前端
    ├── src/                  正式代码
    └── tests/                测试代码(与 src/ 物理分离)
        ├── unit/
        └── integration/
```

**约束5 的物理落实**:测试**不**与正式代码同目录、同文件名后缀混放。
- 后端:测试统一在 `backend/tests/`,**不**在 `app/` 里放 `xxx_test.py`。
- 前端:测试统一在 `frontend/tests/`,**不**在 `src/` 组件旁放 `Xxx.test.tsx`。
- pytest 配置 `testpaths=tests`;vitest 配置 `include: ['tests/**']`,并显式排除 `src/**/*.test.*`。

## 测试时序:严格 TDD(约束5 的核心)

每个任务按 **RED → GREEN → REFACTOR**:

1. **RED**:先在 `tests/` 写测试,运行**应失败**(功能还不存在)。
2. **GREEN**:写 `app/` 或 `src/` 的最小实现让测试通过。
3. **REFACTOR**:重构为生产级代码,测试保持绿。

**禁止**:先把正式代码写完,再回头补测试文件。测试必须先于(或至少不晚于)对应实现存在,且从一开始就在独立 `tests/` 目录。

## 局部测试(约束4)

「每个功能做完之后都进行局部测试」的落实:

- 每完成一个任务(T 号)对应的功能单元,**只跑该功能相关的测试**,不必全量:
  - 后端:`rtk pytest tests/unit/test_<模块>.py`(或按标记 `-m <mark>`)。
  - 前端:`rtk vitest tests/unit/<模块>`。
- 局部测试通过 + 覆盖该功能的关键路径与边界态后,才算该任务完成。
- 提交前再跑一次相关 Epic 的整体测试做回归。
- 覆盖率目标 80%(全局规范),以功能关键路径优先。

## 命令(统一用 rtk 前缀)

```bash
# 后端
rtk pytest tests/unit/test_servers.py     # 局部
rtk pytest tests/ --cov=app               # 全量+覆盖率
# 前端
rtk vitest tests/unit/serverList          # 局部
rtk vitest --coverage                      # 全量+覆盖率
```

## 定义完成(DoD)

一个任务标记完成需同时满足:
- [ ] 先写的测试存在于独立 `tests/` 目录,曾经 RED。
- [ ] 实现让测试 GREEN,局部测试全绿。
- [ ] 覆盖关键路径 + 至少一个边界/失败态。
- [ ] 前端任务额外过「反 AI 感清单」。
- [ ] 代码经 lint/format。
